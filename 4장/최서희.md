# 4장

## 인상 깊은 내용

### 1. 생성형 AI를 잘 활용하려면 무엇이 필요한가?

- **API 게이트웨이**  
  생성형 AI 시스템의 핵심 인프라 구성 요소로, 반복적인 개발 작업을 최소화하고 공통 로직을 중앙에서 관리한다. 이를 통해 시스템의 안정성과 보안성을 함께 향상시킬 수 있다.

- **실험적 피드백**  
  생성형 AI의 생성 결과 품질을 지속적으로 개선하기 위해 사용자 피드백을 수집하고 반영하는 기능이다. 실제 사용 데이터를 기반으로 모델과 프롬프트를 점진적으로 고도화할 수 있다.

- **LLM 서빙**  
  사내 자체 대규모 언어 모델(LLM) 또는 공개된 LLM을 효율적으로 운영·관리·제공하기 위한 영역이다. 모델 배포, 버전 관리, 성능 관리까지 포함한다.

- **프롬프트 실험**  
  효과적인 프롬프트 개발과 최적화를 위한 실험 환경 제공을 의미한다. 다양한 프롬프트를 비교·검증하며 생성 결과의 품질을 높일 수 있다.

- **하이브리드 검색**  
  어휘 검색과 시맨틱 검색을 결합한 고급 검색 방식이다. 키워드 정확성과 의미 기반 검색의 장점을 동시에 활용할 수 있다.

- **RAG 파이프라인**  
  외부 지식을 활용해 생성형 AI의 응답 품질을 향상시키는 구조다. 검색부터 컨텍스트 구성, 응답 생성까지의 파이프라인을 표준화해 일관된 결과를 제공한다.


### 2. 풀어야 할 문제들

- **자격증명 관리**  
  API 호출 시 필요한 키, 토큰 등의 자격증명을 안전하게 관리하고 생성형 AI API에 전달해야 한다. 보안과 직결되는 중요한 문제다.

- **중복 개발 문제**  
  생성형 AI 활용은 외부 API 호출에서 시작되지만,  
  - 개인 식별 정보 처리  
  - 로깅  
  - 호출 제한  
  과 같은 공통 기능이 서비스마다 반복 구현되는 비효율이 발생한다.

- **비효율적인 배포 방식**  
  게이트웨이 도입 이전에는 자격증명, 프롬프트, 모델 배포명, API 버전, 세부 파라미터 등을 코드나 리소스 파일로 직접 관리해야 했다. 이 방식은 변경에 취약하고 운영 부담을 증가시킨다.
